# Code of conduct

## The short version

Be kind. Assume good faith. Focus on the work. Don't be a jerk to anyone for any reason.

## The slightly longer version

This project welcomes contributions from anyone, regardless of experience, background, identity, or affiliation. The only thing we ask in return is that everyone — contributors, maintainers, users, and bystanders — treat each other with basic decency.

### Expected behavior

- Be respectful in disagreements. Critique ideas, not people.
- Assume the person you're talking to is acting in good faith until they give you concrete reason to believe otherwise.
- When you don't understand a contribution or a comment, ask before attacking.
- Acknowledge when you were wrong. It is not weakness; it is how the work gets better.
- Help newcomers. Everyone was new once.

### Unacceptable behavior

- Personal attacks, harassment, or insults of any kind.
- Discrimination or derogatory comments based on identity, background, or affiliation.
- Sustained disruption of discussions or reviews.
- Publishing others' private information without consent.
- Sexual attention or imagery in project spaces.
- Threats of violence or wishing harm on anyone.
- Any conduct that would reasonably be considered inappropriate in a professional setting.

This applies in every project space: issues, pull requests, discussions, code review, commit messages, the project's chat (if one exists), and any conversation that publicly identifies itself as being about this project.

## Reporting

If you experience or witness behavior that violates this code, please report it privately to the maintainers via email (see the maintainer's GitHub profile for the address). Reports will be handled confidentially.

When reporting, include:

- What happened, where, and when.
- Who was involved.
- Any links, screenshots, or context that would help us understand.
- Whether you want to be kept informed of the outcome.

## Enforcement

Maintainers will review every report and respond as quickly as possible — within a few days at the outside. Possible responses include, in roughly increasing severity:

1. A private clarification or reminder.
2. A public clarification or reminder.
3. A request that specific behavior stop.
4. A temporary ban from project spaces.
5. A permanent ban from project spaces.

The response is at the maintainers' discretion and depends on the nature, severity, and pattern of the behavior. We will favor the lightest response that resolves the situation, and we will not hesitate to escalate when the situation warrants it.

Maintainers who violate this code are held to the same standard as anyone else, with the additional expectation that they should know better.

## Attribution

This code of conduct draws on the spirit of the [Contributor Covenant](https://www.contributor-covenant.org/) and the [Go Community Code of Conduct](https://golang.org/conduct), with the goal of being short enough that everyone actually reads it.
